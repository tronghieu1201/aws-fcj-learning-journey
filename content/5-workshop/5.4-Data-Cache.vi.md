## Amazon RDS PostgreSQL (Multi-AZ)

| Thông số | Giá trị |
| --- | --- |
| Engine | PostgreSQL 16 |
| Instance class | `db.t3.micro` |
| Multi-AZ | Bật (Primary + Standby, đồng bộ tự động) |
| Subnet Group | Trải 2 Private Subnet (Data tier), 2 AZ |
| Kết nối | Chỉ chấp nhận từ `ec2-sg` và `lambda-sg` qua port 5432 |
| Mã hóa | `storage encrypted` bằng AWS KMS Customer Managed Key |
| SSL | Bắt buộc (`force_ssl=1`), kết nối từ ứng dụng dùng `ssl: { rejectUnauthorized: false }` |

Ứng dụng (EC2 API và Lambda Worker) chỉ kết nối tới **endpoint của RDS Primary** — không có kết nối trực tiếp nào tới Standby. Khi Primary gặp sự cố, AWS tự động chuyển hướng DNS endpoint sang Standby (failover), ứng dụng không cần biết Standby tồn tại.

Schema quản lý bằng Prisma ORM, 11 bảng, migrate qua `npx prisma migrate deploy` — chạy từ xa bằng SSM Session Manager port-forwarding (không mở port 5432 ra Internet).

## ElastiCache Redis

| Thông số | Giá trị |
| --- | --- |
| Engine | Redis 7.x |
| Node type | `cache.t3.micro` |
| Vị trí | Private Subnet (Data tier), Security Group `redis-sg` (chỉ nhận từ `ec2-sg`, port 6379) |
| Dùng cho | Cache Brand Persona (TTL 30 phút), rate limit theo tổ chức |

Redis chỉ được truy cập bởi **EC2 (Express API)** — Lambda Worker không kết nối tới Redis, vì tác vụ AI chạy bất đồng bộ không cần cache persona (đọc thẳng từ RDS khi xử lý job).

## AWS Secrets Manager

Toàn bộ secret nhạy cảm (JWT secret, database credentials, Gemini API key) được gộp vào **1 secret JSON bundle** duy nhất (`quillo/app-secrets-prod`), thay vì đọc trực tiếp từ file `.env` trên production.

```bash
USE_SECRETS_MANAGER=true   # production
USE_SECRETS_MANAGER=false  # local dev, đọc trực tiếp từ .env
```

Cơ chế nạp secret (`loadSecrets()`) chạy **trước** khi các module khác (database, JWT, AI provider) được import — tránh race condition đọc `process.env` trước khi secret kịp nạp vào.

EC2 và Lambda dùng IAM Role riêng (`quillo-ec2-role`, `quillo-lambda-role`) để đọc secret — không có access key tĩnh nào được lưu trên instance hay trong code.

## Kiểm chứng

```bash
aws rds describe-db-instances --db-instance-identifier quillo-prod-db --region ap-southeast-1 \
  --query "DBInstances[0].[MultiAZ,DBInstanceStatus,Endpoint.Address]"

aws elasticache describe-cache-clusters --cache-cluster-id quillo-redis-prod --region ap-southeast-1

aws secretsmanager get-secret-value --secret-id quillo/app-secrets-prod --region ap-southeast-1 \
  --query "SecretString" --output text | jq 'keys'
```

> **Kết quả mong đợi:** lệnh RDS trả về `true` (Multi-AZ) và `"available"`; lệnh ElastiCache trả về `"CacheClusterStatus": "available"`; lệnh Secrets Manager trả về đúng 3 field `["DATABASE_URL", "GEMINI_API_KEY", "JWT_SECRET"]` (không hiện giá trị thật).
