## Đóng gói ứng dụng với Docker

Backend Express API được đóng gói bằng Dockerfile **multi-stage**:

| Stage | Vai trò |
| --- | --- |
| `builder` | Cài dependencies, chạy `prisma generate`, biên dịch TypeScript (`tsc`) |
| `runtime` | Image `node:20-slim`, chỉ copy `dist/` + `node_modules` cần thiết, chạy bằng **non-root user**, có `HEALTHCHECK` |

Image build theo `--platform linux/amd64` (tránh lỗi kiến trúc arm64/amd64 khi build trên máy Apple Intel/Silicon), push lên **Amazon ECR** (`quillo-api`, bật `scanOnPush`).

## Application Load Balancer

| Thông số | Giá trị |
| --- | --- |
| Loại | Internet-facing, đặt tại Public Subnet |
| Listener | HTTP:80 → Target Group |
| Target Group | Port 3001, protocol HTTP |
| Health Check | `/api/v1/health`, threshold mặc định |
| Security Group | `alb-sg` (80/443 từ `0.0.0.0/0`) |

## Auto Scaling Group

| Thông số | Giá trị |
| --- | --- |
| Launch Template | Amazon Linux 2023, `t3.small`, user-data pull image từ ECR và chạy container lúc boot |
| Min / Max | 2 / 4 |
| Trải AZ | 1 instance/AZ tối thiểu (2 Private App Subnet) |
| Scaling Policy | Target Tracking theo CPU trung bình 60% |
| Health Check | ELB health check (không chỉ EC2 status check) — instance unhealthy theo ALB sẽ bị thay thế |

Khi cần deploy phiên bản mới: CI/CD gọi `autoscaling start-instance-refresh`, ASG lần lượt thay từng instance bằng bản mới, không downtime nhờ ALB vẫn phục vụ từ các instance còn lại trong lúc refresh.

## Kiểm chứng

```bash
aws elbv2 describe-target-health --target-group-arn <tg-arn> --region ap-southeast-1
aws autoscaling describe-auto-scaling-groups --auto-scaling-group-names quillo-api-asg --region ap-southeast-1 \
  --query "AutoScalingGroups[0].[MinSize,MaxSize,Instances[].HealthStatus]"
curl -I http://<alb-dns-name>/api/v1/health
```

> **Kết quả mong đợi:** `describe-target-health` trả về toàn bộ target ở trạng thái `"State": "healthy"`; ASG trả về `MinSize=2`, mọi instance có `HealthStatus: "Healthy"`; `curl` trả về `HTTP/1.1 200 OK`.
