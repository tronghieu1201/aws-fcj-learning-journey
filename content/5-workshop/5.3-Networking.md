## VPC Design

All infrastructure runs inside a dedicated VPC (not the Default VPC), spanning two Availability Zones for high availability:

| Component | Detail |
| --- | --- |
| VPC | `10.0.0.0/16`, region `ap-southeast-1` |
| Availability Zones | `ap-southeast-1a`, `ap-southeast-1b` |
| Public Subnet (2) | Hosts the Application Load Balancer + NAT Gateway (1/AZ) |
| Private Subnet — App (2) | Hosts EC2 (Auto Scaling Group) and the Lambda Worker (ENI) |
| Private Subnet — Data (2) | Hosts RDS Multi-AZ and ElastiCache Redis |
| Internet Gateway | 1, attached at the VPC level, used only by the NAT Gateways/ALB |

Deployed via the `infrastructure/scripts/setup-vpc.sh` script: creates the VPC, subnets, route tables, Internet Gateway, NAT Gateways, and 4 corresponding Security Groups.

## Security Groups

| Security Group | Allowed Inbound |
| --- | --- |
| `alb-sg` | HTTP/HTTPS (80/443) from `0.0.0.0/0` |
| `ec2-sg` | Port 3001, only from `alb-sg` |
| `rds-sg` | Port 5432, only from `ec2-sg` and `lambda-sg` |
| `redis-sg` | Port 6379, only from `ec2-sg` |
| `lambda-sg` | No inbound (Lambda only initiates outbound calls) |

> Principle: each tier only opens the port it needs, sourced only from the adjacent tier's Security Group — never from static IP ranges.

## IAM Roles

| Role | Attached to | Main permissions |
| --- | --- | --- |
| `quillo-ec2-role` | EC2 instance (Instance Profile) | Secrets Manager (read), SQS (send), S3 (put/get), CloudWatch Logs |
| `quillo-lambda-role` | Lambda Worker | Secrets Manager (read), SQS (consume + delete message), S3 (put/get), RDS (via Security Group, not IAM) |
| `quillo-github-actions-deploy-role` | GitHub Actions (via OIDC, no access keys) | ECR push, scoped `autoscaling:StartInstanceRefresh`, Lambda `UpdateFunctionCode`, S3 sync |

Lambda and EC2 use **separate** IAM Roles (not shared) under the least-privilege principle — limiting the blast radius if either role's permissions were ever compromised.

## Verification

```bash
aws ec2 describe-vpcs --filters "Name=tag:Project,Values=quillo" --region ap-southeast-1
aws ec2 describe-subnets --filters "Name=vpc-id,Values=<vpc-id>" --region ap-southeast-1
aws iam list-attached-role-policies --role-name quillo-ec2-role
```

> **Expected result:** `describe-vpcs` returns `"State": "available"`; `describe-subnets` returns 6 subnets with `"AvailabilityZone"` split evenly across `ap-southeast-1a`/`ap-southeast-1b`; `list-attached-role-policies` returns a non-empty list of `PolicyName`.
