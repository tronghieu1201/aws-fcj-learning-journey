## Thiết kế VPC

Toàn bộ hạ tầng nằm trong 1 VPC riêng (không dùng Default VPC), trải 2 Availability Zone để đảm bảo tính sẵn sàng cao:

| Thành phần | Chi tiết |
| --- | --- |
| VPC | `10.0.0.0/16`, region `ap-southeast-1` |
| Availability Zones | `ap-southeast-1a`, `ap-southeast-1b` |
| Public Subnet (2) | Chứa Application Load Balancer + NAT Gateway (1/AZ) |
| Private Subnet — App (2) | Chứa EC2 (Auto Scaling Group) và Lambda Worker (ENI) |
| Private Subnet — Data (2) | Chứa RDS Multi-AZ và ElastiCache Redis |
| Internet Gateway | 1, gắn ở cấp VPC, chỉ phục vụ NAT Gateway/ALB |

Triển khai bằng script `infrastructure/scripts/setup-vpc.sh`: tạo VPC, Subnet, Route Table, Internet Gateway, NAT Gateway và 4 Security Group tương ứng.

## Security Groups

| Security Group | Inbound cho phép |
| --- | --- |
| `alb-sg` | HTTP/HTTPS (80/443) từ `0.0.0.0/0` |
| `ec2-sg` | Port 3001 chỉ từ `alb-sg` |
| `rds-sg` | Port 5432 chỉ từ `ec2-sg` và `lambda-sg` |
| `redis-sg` | Port 6379 chỉ từ `ec2-sg` |
| `lambda-sg` | Không có inbound (Lambda chỉ gọi ra ngoài) |

> Nguyên tắc: mỗi tầng chỉ mở đúng port cần thiết, chỉ cho phép nguồn là Security Group của tầng liền kề — không mở theo dải IP tĩnh.

## IAM Roles

| Role | Gắn cho | Quyền chính |
| --- | --- | --- |
| `quillo-ec2-role` | EC2 instance (Instance Profile) | Secrets Manager (đọc), SQS (send), S3 (put/get), CloudWatch Logs |
| `quillo-lambda-role` | Lambda Worker | Secrets Manager (đọc), SQS (consume + delete message), S3 (put/get), RDS (qua Security Group, không qua IAM) |
| `quillo-github-actions-deploy-role` | GitHub Actions (qua OIDC, không dùng access key) | ECR push, `autoscaling:StartInstanceRefresh` (scoped ARN), Lambda `UpdateFunctionCode`, S3 sync |

Lambda và EC2 dùng 2 Role **tách biệt** (không dùng chung) theo nguyên tắc least-privilege — giảm phạm vi ảnh hưởng nếu 1 trong 2 bị lộ quyền.

## Kiểm chứng

```bash
aws ec2 describe-vpcs --filters "Name=tag:Project,Values=quillo" --region ap-southeast-1
aws ec2 describe-subnets --filters "Name=vpc-id,Values=<vpc-id>" --region ap-southeast-1
aws iam list-attached-role-policies --role-name quillo-ec2-role
```

> **Kết quả mong đợi:** `describe-vpcs` trả về `"State": "available"`; `describe-subnets` trả về 6 subnet với `"AvailabilityZone"` chia đều `ap-southeast-1a`/`ap-southeast-1b`; `list-attached-role-policies` trả về danh sách `PolicyName` không rỗng.
