## Kiểm thử luồng nghiệp vụ end-to-end

Thực hiện kiểm thử thủ công toàn bộ luồng chính trên domain production thật (`https://quillo.khuongle.site`), không phải môi trường local/staging:

| Bước | Hành động | Kết quả kỳ vọng |
| --- | --- | --- |
| 1 | Đăng ký tổ chức mới (`POST /auth/register`) | Tạo `Organization` + `User` role `OWNER`, tự động đăng nhập |
| 2 | Tạo Brand Persona (tone, keyword, đối tượng mục tiêu) | Persona lưu vào RDS, cache vào Redis (TTL 30 phút) |
| 3 | Tạo Content Piece, chọn Persona, nhập brief, bấm Generate | Nhận `202 Accepted` + `jobId` ngay lập tức (không chờ AI) |
| 4 | Frontend poll `GET /content/:id/jobs/:jobId` mỗi 2.5s | Job chuyển `QUEUED` → `PROCESSING` → `COMPLETED`, nội dung AI hiển thị |
| 5 | Export nội dung ra PDF | File có font tiếng Việt hiển thị đúng (Noto Sans), tải qua presigned URL |
| 6 | Kiểm tra trang Usage | Token đã dùng tăng đúng theo lần generate vừa thực hiện |

## Kiểm thử Quota Enforcement

Tạo liên tục nhiều request generate cho tới khi vượt `monthlyTokenQuota` của tổ chức test — xác nhận:

- API trả về `429` với message `QUOTA_EXCEEDED` **trước khi** message được đẩy vào SQS (tránh lãng phí tài nguyên worker cho request chắc chắn bị từ chối).
- Worker cũng tự kiểm tra lại quota trước khi gọi AI Provider — nếu vượt, đánh dấu job `FAILED` và return, **không throw exception** (tránh SQS coi là lỗi tạm thời rồi tự động retry, dồn ứ hàng đợi vô ích).

## Vận hành hàng ngày (Operational Runbook)

| Việc kiểm tra | Lệnh / nơi xem |
| --- | --- |
| Health check nhanh | `curl -f https://quillo-api.khuongle.site/api/v1/health` |
| Trạng thái ASG | CloudWatch Alarms (mục 5.7) — 3 alarm nghiệp vụ đều phải `OK` |
| Log lỗi gần nhất | CloudWatch Logs Insights, log group `/quillo/api` hoặc `/quillo/worker` |
| DLQ có message tồn đọng | `aws sqs get-queue-attributes --queue-url <dlq-url> --attribute-names ApproximateNumberOfMessages` — khác 0 nghĩa là có job xử lý thất bại quá 5 lần, cần điều tra thủ công |
| Chi phí phát sinh bất thường | AWS Cost Explorer, lọc theo tag `Project=quillo` |

## Kiểm chứng

```bash
curl -w "\n%{http_code}\n" -X POST https://quillo-api.khuongle.site/api/v1/content/<id>/generate \
  -H "Authorization: Bearer <token>"

aws sqs get-queue-attributes --queue-url <dlq-url> --attribute-names ApproximateNumberOfMessages
```

> **Kết quả mong đợi:** request generate hợp lệ trả về `202` kèm `jobId`; request vượt quota trả về `429`; `ApproximateNumberOfMessages` của DLQ bằng `0` (không có job nào thất bại tồn đọng).
