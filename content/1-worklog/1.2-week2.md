## 3. Content

- AWS Networking:
  - Learned about basic networking components in AWS, including Amazon VPC, subnet, route table, and Internet Gateway.
  - Practiced creating a VPC and enabling DNS resolution and DNS hostnames.
  - Configured subnets, route tables, and Internet Gateway to build a basic network model.
  - Used VPC Resource Map to observe the overall network resources.

- IAM Role:
  - Learned how to grant EC2 access to AWS services using IAM Role.
  - Compared using direct Access Key with using IAM Role.
  - Practiced creating an IAM Role, attaching an S3 policy, and assigning the role to an EC2 instance.

- Route 53:
  - Learned about the Hybrid DNS model between AWS environment and internal/on-premises DNS systems.
  - Deployed lab infrastructure using CloudFormation.
  - Created Route 53 Resolver Outbound Endpoint, Inbound Endpoint, and Resolver Rule.
  - Verified DNS resolution using `nslookup` and checked connectivity using `ping`.

- AWS CLI:
  - Learned how to use AWS CLI to interact with AWS services through the command line.
  - Checked AWS CLI on EC2 and verified that credentials were retrieved from IAM Role.
  - Worked with S3, SNS, IAM, VPC, and EC2 using command line commands.
  - Cleaned up resources after completing the lab.
