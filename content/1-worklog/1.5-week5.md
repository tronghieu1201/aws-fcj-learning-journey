## 3. Content

- Amazon S3 Security:
  - Created the lab environment using CloudFormation for the S3 Security hands-on lab.
  - Adjusted the template to use the EC2 `t3.micro` instance type, which was suitable for the practice account.
  - Connected to the EC2 lab instance using EC2 Instance Connect and worked with the AWS CLI.
  - Created access keys for `SID-lab-user1` and `SID-lab-user2`, then configured the CLI profiles `user1` and `user2`.
  - Checked access permissions for the S3 Security Lab bucket and the sample data folders `app1/` and `app2/`.
  - Configured a bucket policy to require S3 access through HTTPS.
  - Cleaned up resources after completing the lab.

- IAM Cost Control:
  - Learned how to use IAM policies to limit AWS resource usage.
  - Practiced restricting resource creation based on Region, EC2 instance type, and EBS volume type.
  - Became familiar with IAM Conditions to control costs and prevent resources from being created outside the allowed scope.

- Tags and Resource Groups:
  - Learned how to use Tags to classify AWS resources by environment, owner, or purpose.
  - Practiced adding tags, filtering resources by tags, and managing resources by group.
  - Learned how Resource Groups can be used to group resources with the same tags, making management and monitoring easier.
