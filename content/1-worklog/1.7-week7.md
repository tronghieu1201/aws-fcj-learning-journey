## 3. Topics Covered

*   **IAM Permission Boundaries:**
    *   Wrote boundary policies to limit the maximum permissions delegated administrators can assign.
    *   Validated authorization rules on test accounts to ensure privilege escalation was blocked.
*   **AssumeRole Condition Policy:**
    *   Wrote conditional keys in IAM Trust Relationships to limit Role assumption.
    *   Configured filters restricting access to corporate IP blocks and active business hours.
*   **AWS Security Hub Assessments:**
    *   Activated Security Hub to run automated security compliance scans against target cloud accounts.
    *   Reviewed score breakdowns against standard security frameworks, including CIS and AWS Foundations.
*   **AWS WAF Web Application Firewalls:**
    *   Created Web Access Control Lists (Web ACLs) and attached them to Application Load Balancers.
    *   Configured Managed Rulesets to block common exploits, including Cross-Site Scripting (XSS) and SQL injection.
*   **Personal Project Access Audit:**
    *   Applied the Principle of Least Privilege to the portfolio Lambda execution roles.
    *   Removed unnecessary wildcard resource access permissions (`"Resource": "*"`) from the custom DynamoDB policies.
