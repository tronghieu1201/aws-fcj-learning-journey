## 3. Content

- AWS Lambda and Slack:
  - Learned how to use AWS Lambda to automate EC2 operational tasks.
  - Created an IAM Role that grants Lambda permissions to work with EC2 and CloudWatch Logs.
  - Added tags to EC2 instances so Lambda could identify the correct instances to start or stop.
  - Configured a Lambda function to automatically start and stop EC2 instances based on usage requirements.
  - Integrated a Slack Webhook to send notifications when Lambda completed an operation successfully.

- CloudWatch and Grafana:
  - Learned how to monitor AWS resources through CloudWatch metrics.
  - Created an EC2 instance and configured an IAM Role so Grafana could read monitoring data.
  - Installed Grafana on EC2 and connected Grafana to CloudWatch as a data source.
  - Created a basic dashboard to monitor CPU, network, and resource status.
  - Became familiar with visualizing monitoring data instead of only viewing metrics directly in the AWS Console.

- AWS Systems Manager Session Manager:
  - Learned how to access EC2 securely using Session Manager without an SSH key.
  - Configured an IAM Role for EC2 to use AWS Systems Manager.
  - Connected to public and private EC2 instances through Session Manager in the AWS Console.
  - Understood the security benefit of Session Manager because it reduces the need to open SSH port 22.
  - Checked session logs to monitor access history and activities on the instance.

- AWS CloudFormation:
  - Learned the concept of Infrastructure as Code in AWS infrastructure deployment.
  - Practiced writing and using a CloudFormation template to create resources automatically.
  - Created a CloudFormation Stack, checked the deployment status, and reviewed the list of created resources.
  - Updated and deleted stacks to manage the lifecycle of AWS resources.
  - Understood how CloudFormation helps deploy infrastructure in a consistent, repeatable, and manageable way.
