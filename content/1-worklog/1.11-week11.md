## 3. Topics Covered

*   **Designing the Portfolio Architecture Diagram:**
    *   Designed a clean architecture diagram using Draw.io to represent actual deployed cloud services.
    *   Excluded Route 53 from the diagram as the project uses the default CloudFront HTTPS domain instead of a custom domain.
    *   Represented actual traffic paths: User -> CloudFront CDN -> S3 (Static Assets) and client AJAX -> API Gateway -> Lambda -> DynamoDB.
*   **System-wide End-to-End Testing:**
    *   Accessed the live site via the CloudFront URL and verified fast, secure asset loads over HTTPS.
    *   Verified that reloading pages did not return S3 `AccessDenied` errors.
    *   Tested the `/visitor` endpoint directly and verified that DB count records incremented properly.
*   **Drafting Technical Workshop Documentation:**
    *   Drafted the **Overview** section describing project goals and benefits.

    *   Drafted the **Deployment** section detailing Vite builds and S3 asset uploads.
    *   Drafted the **Testing, Monitoring, and Clean-Up** sections to guide users through testing flows, inspecting Lambda execution logs in CloudWatch, and cleaning up resources to prevent unexpected charges.
