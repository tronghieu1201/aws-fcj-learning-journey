Key Highlights

Non-Determinism of “Deterministic” LLM Settings (Speaker: Duc Dao - Cloud Kinetics)

  • Token Selection Mechanism: LLMs generate text token by token by converting logits into probabilities via a softmax function. Theoretically, setting Temperature = 0 triggers greedy decoding (argmax) — always selecting the single highest-probability token to guarantee 100% reproducible and deterministic outputs for the same prompt.
  • The Reality Gap: In practice, even with temp=0, LLMs can still return non-deterministic responses. This randomness does not stem from sampling, but rather from the underlying GPU architecture, floating-point calculation discrepancies, and various inference optimization algorithms utilized during hosting.
  • Impacts & Mitigation: Setting temp=0 often leads the model to get stuck in repetitive generation loops. A practical remedy is setting temp=0.1 as a “sweet spot”, which keeps the probability distribution extremely sharp (near-deterministic) while introducing a tiny stochastic nudge to prevent loop locks. Additionally, engineers should leverage higher repeat penalty values and fundamentally design downstream applications to tolerate variance from the start.

Enterprise-Grade Multi-Agent System: The Case of Startup Credit Scoring (Speaker: Vy Lam - VPBank)

  • The Structural Mismatch: Traditional banking credit scoring systems routinely fail to evaluate startups due to data misalignment. Traditional frameworks mandate 3+ years of audited financial statements, stable collateral, and predictable revenue patterns. In contrast, startup data is highly multi-dimensional, heavily unstructured, and volatile (6-18 months of operation, with core assets being IP, team composition, and market traction).
  • The Virtual Credit Committee Blueprint: To tackle multi-dimensional data, the architecture shifts from a Single Agent to a Multi-Agent paradigm. The system orchestrates multiple highly specialized agents: a Financial Agent (evaluating burn rate and runway), a Market Agent (analyzing TAM/SAM), a Team Agent (vetting founder backgrounds), and a Traction Agent. This distributed expertise delivers comprehensive insights and robust auditability that a single agent cannot match.
  • Enterprise Mindset & ROI: Transitioning AI from a working prototype to production-ready requires security and compliance to be implemented as a Day 1 mindset. The production architecture incorporates Cognito/OAuth2 for authentication, Bedrock Guardrails for content filtering, IAM least-privilege boundaries, and Model Context Protocol (MCP) integration, all deployed via Terraform. This delivers a compelling business case with a 90%+ cost reduction and 95% faster processing speed.

  • LLM Principles: Gained a deeper understanding of model non-determinism and the critical role of hyperparameter tuning in production environments.
  • Multi-Agent Architectures: Discovered how coordinating multiple specialized AI agents can systematically solve complex, domain-specific enterprise problems.
  • Security-First Integration: Realized that embedding security layers (Cognito, Bedrock Guardrails, least-privilege IAM) must be a Day 1 design priority.

Practical Skills

  • Learned how to optimize compute and cost margins when deploying AI systems at scale.
  • Discovered how to bridge academic theories with real-world enterprise applications in corporate finance.

Application to Work

  • Applying the Context Framework (Goals, Constraints, Criteria) to systematically interact with AI tools, dramatically improving daily task efficiency.
  • Strictly adhering to rigid Cloud IAM access controls and security boundary standards during upcoming enterprise internship assignments.

Personal Experience

Highlights

  • Strongly impressed by the Virtual Credit Committee architecture that elegantly solves the lack of historical startup financials.

Key Takeaways

  • The future does not pit Humans against AI, but rather humans who know how to engineer context against those who do not.
  • When pushing AI into production, stability, security boundaries, and data auditability hold far more weight than a flashy proof-of-concept demo.
