3. Nội dung

  • Amazon EC2:
    • Tạo EC2 Windows và Linux, cấu hình Security Group, kết nối bằng RDP/SSH và kiểm tra trạng thái hoạt động.
    • Thực hành EBS Snapshot, Custom AMI và tạo instance mới từ AMI.
    • Cấu hình IAM Role và sử dụng Systems Manager Session Manager để truy cập EC2-Windows trong tình huống mất Key Pair.
  • AWS Cloud9 / Kiro CLI:
    • Ghi nhận AWS Cloud9 không khả dụng với tài khoản AWS mới.
    • Cài đặt, đăng nhập và sử dụng Kiro CLI trong VS Code Terminal để tạo project Node.js mẫu.
  • Amazon Lightsail:
    • Triển khai WordPress và PrestaShop trên Amazon Lightsail.
    • Cấu hình Lightsail Database, Static IP và kiểm tra website qua trình duyệt.
  • Auto Scaling:
    • Triển khai mô hình ứng dụng EC2 có Load Balancer và Auto Scaling.
    • Cấu hình Launch Template, Target Group, ALB và Auto Scaling Group.
    • Kiểm thử truy cập ứng dụng thông qua Load Balancer DNS.

4. Kết quả đạt được

  • Tạo và kết nối thành công EC2 Windows bằng RDP, EC2 Linux bằng SSH.
  • Thực hành được Snapshot, Custom AMI và tạo instance mới từ AMI.
  • Truy cập được EC2-Windows bằng AWS Systems Manager khi mất Key Pair.
  • Cài đặt và đăng nhập thành công Kiro CLI.
  • Tạo và chạy được project Node.js mẫu bằng Kiro CLI.
  • Tạo được database, instance và Static IP trên Lightsail.
  • Truy cập được WordPress và PrestaShop.
  • Đăng nhập được trang quản trị của các website đã triển khai.
  • Tạo được Launch Template, ALB, Target Group và Auto Scaling Group.
  • Truy cập được ứng dụng qua Load Balancer DNS.
  • Nắm được quy trình tự động mở rộng EC2 bằng Auto Scaling.
  • Dọn dẹp tài nguyên sau khi hoàn thành lab để tránh phát sinh chi phí.
