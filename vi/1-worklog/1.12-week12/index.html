3. Nội dung

  • Nhìn lại giai đoạn đầu của quá trình thực tập:
    • Tuần 1 giúp em làm quen với tài khoản AWS, giao diện AWS Console, Free Tier, Billing, AWS Budgets, AWS Support và IAM.
    • Tuần 2 giúp em hiểu rõ hơn về mạng trên AWS thông qua VPC, subnet, route table, Internet Gateway, IAM Role cho EC2, Route 53 và AWS CLI.
    • Tuần 3 giúp em thực hành nhóm dịch vụ compute như EC2, Lightsail, Launch Template, Load Balancer và Auto Scaling Group.
    • Tuần 4 giúp em học thêm về S3 static website hosting, RDS database, CloudWatch monitoring và bắt đầu định hướng project cá nhân.
  • Nhìn lại giai đoạn giữa của quá trình thực tập:
    • Tuần 5 tập trung vào bảo mật S3, kiểm soát chi phí bằng IAM, quản lý tài nguyên bằng tag và bắt đầu tạo backend serverless cho visitor counter bằng DynamoDB và Lambda.
    • Tuần 6 giúp em hiểu cách dùng Lambda để tự động hóa, CloudWatch để giám sát, Systems Manager để truy cập EC2 an toàn, CloudFormation cho Infrastructure as Code và API Gateway để kết nối frontend với Lambda.
    • Tuần 7 giúp em có góc nhìn rõ hơn về bảo mật trên AWS, gồm Permission Boundary, IAM Condition, Security Hub, WAF và rà soát quyền cho project cá nhân.
    • Tuần 8 giúp em tiếp cận thêm các chủ đề reliability, backup, Docker, ECS và chỉnh sửa source portfolio để build bản production.
  • Nhìn lại giai đoạn hoàn thiện project:
    • Tuần 9 tập trung deploy portfolio lên Amazon S3 và phân phối website qua CloudFront.
    • Tuần 10 tập trung chỉnh sửa source, cập nhật README, kiểm tra lại CloudFront, xử lý lỗi refresh trang và chuẩn hóa project trước khi làm tài liệu.
    • Tuần 11 tập trung hoàn thiện sơ đồ kiến trúc, kiểm thử luồng end-to-end và viết documentation cho AWS Cloud Portfolio.
  • Tổng kết project cá nhân AWS Cloud Portfolio:
    • Project đã được triển khai thành website portfolio chạy qua CloudFront URL.
    • Frontend được build bằng Vue, TypeScript và Vite.
    • File build frontend được upload lên Amazon S3.
    • CloudFront được dùng để phân phối website qua HTTPS mặc định.
    • Visitor counter được xây dựng bằng API Gateway, Lambda và DynamoDB.
    • CloudWatch Logs được dùng để kiểm tra quá trình Lambda chạy.
    • IAM Role được dùng để cấp quyền cho Lambda thao tác với DynamoDB và ghi log.
    • Sơ đồ kiến trúc và documentation đã được bổ sung để giải thích rõ cách project hoạt động.
  • Kiến thức chính học được sau 12 tuần:
    • Hiểu được cách tổ chức tài khoản AWS và kiểm soát chi phí cơ bản.
    • Biết cách dùng IAM để quản lý quyền truy cập và hạn chế quyền theo nguyên tắc least privilege.
    • Hiểu các thành phần mạng cơ bản trong AWS như VPC, subnet, route table, Internet Gateway và security group.
    • Biết cách triển khai compute bằng EC2, Lightsail và Auto Scaling.
    • Biết cách dùng S3 để lưu trữ file tĩnh và host website.
    • Biết cách dùng RDS và DynamoDB cho hai hướng database khác nhau: relational database và NoSQL serverless database.
    • Biết cách tạo API bằng API Gateway và xử lý backend bằng Lambda.
    • Biết cách dùng CloudWatch để xem log, theo dõi lỗi và kiểm tra quá trình thực thi.
    • Biết cách dùng CloudFront để phân phối website tĩnh ổn định hơn.
    • Hiểu hơn về bảo mật, cleanup tài nguyên và tránh phát sinh chi phí không cần thiết.
  • Bài học kinh nghiệm:
    • Khi làm AWS cần chú ý region, vì nhiều lỗi xảy ra do kiểm tra sai region.
    • Cần kiểm tra billing thường xuyên, đặc biệt sau khi tạo EC2, RDS, NAT Gateway, WAF hoặc các dịch vụ có thể phát sinh chi phí.
    • Không nên ghi dịch vụ vào báo cáo nếu project chưa thật sự dùng, ví dụ Route 53 hoặc custom domain.
    • Source code nên lưu trên GitHub, còn S3 chỉ nên chứa file build frontend.
    • Khi deploy website SPA lên S3 và CloudFront cần chú ý lỗi refresh trang và cấu hình error page phù hợp.
    • Khi xây dựng backend serverless, cần kiểm tra đủ cả API Gateway, Lambda, DynamoDB và CloudWatch Logs để chắc chắn luồng chạy đúng.
    • Việc viết documentation và sơ đồ kiến trúc giúp project dễ hiểu hơn, không chỉ có kết quả chạy được mà còn giải thích được cách hệ thống hoạt động.
  • Đánh giá cá nhân:
    • Sau 12 tuần, em đã có cái nhìn rõ hơn về cách các dịch vụ AWS kết nối với nhau trong một project thực tế.
    • Ban đầu em chỉ làm theo từng lab riêng lẻ, nhưng về sau đã biết cách ghép các dịch vụ lại thành một project hoàn chỉnh.
    • Project AWS Cloud Portfolio giúp em hiểu rõ hơn về static hosting, CDN, API serverless, database serverless, log monitoring và kiểm soát chi phí.
    • Phần còn hạn chế là một số dịch vụ nâng cao như ECS, Step Functions, Cognito, SQS/SNS và X-Ray mới dừng ở mức làm quen, chưa áp dụng sâu vào project cá nhân.
    • Sau kỳ thực tập, em cần tiếp tục luyện thêm về CI/CD, Infrastructure as Code, bảo mật IAM nâng cao và triển khai project có custom domain hoàn chỉnh.

4. Kết quả đạt được

  • Tổng hợp lại được nội dung đã học trong 11 tuần trước.
  • Nhìn lại được quá trình hình thành và hoàn thiện AWS Cloud Portfolio.
  • Xác định được các nhóm kiến thức chính đã học: IAM, Networking, Compute, Storage, Database, Serverless, Security, Monitoring và Cost Control.
  • Rút ra được kinh nghiệm khi triển khai website tĩnh trên S3 và CloudFront.
  • Rút ra được kinh nghiệm khi xây dựng visitor counter bằng API Gateway, Lambda và DynamoDB.
  • Hiểu rõ hơn vai trò của documentation và architecture diagram trong một project AWS.
  • Nhận diện được các điểm cần học tiếp sau thực tập như CI/CD, Infrastructure as Code, custom domain và security nâng cao.
  • Hoàn thiện nội dung tổng kết cuối kỳ cho báo cáo thực tập.
